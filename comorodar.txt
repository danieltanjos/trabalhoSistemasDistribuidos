COMO RODAR O PROJETO

Abrir o terminal na pasta do projeto.

1. Baixar o Maven local

Comandos:

curl -L https://archive.apache.org/dist/maven/maven-3/3.9.11/binaries/apache-maven-3.9.11-bin.tar.gz -o /tmp/apache-maven-3.9.11-bin.tar.gz
tar -xzf /tmp/apache-maven-3.9.11-bin.tar.gz -C /tmp


2. Compilar o projeto Java

Comando:

/tmp/apache-maven-3.9.11/bin/mvn -Dmaven.repo.local=/tmp/m2repo -f java-core/pom.xml compile


3. Montar o classpath do Java

Comando:

export JAVA_CORE_CP="java-core/target/classes:$(find /tmp/m2repo -name '*.jar' | tr '\n' ':')"


4. Rodar tudo com um comando

Comando:

java -cp "$JAVA_CORE_CP" br.senai.corretora.DemoMain

Esse comando sobe:

- custodia gRPC
- core da corretora
- gateway TCP
- simulador de corretoras
- consulta final do painel RMI

5. Rodar manualmente, se quiser separar os modulos

Abrir 5 terminais separados.

Terminal 1: Custodia

java -cp "$JAVA_CORE_CP" br.senai.corretora.CustodyServerMain

Terminal 2: Core

java -cp "$JAVA_CORE_CP" br.senai.corretora.BrokerServerMain

Terminal 3: Gateway

java -cp "$JAVA_CORE_CP" br.senai.corretora.GatewayServerMain

Terminal 4: Simulador

java -cp "$JAVA_CORE_CP" br.senai.corretora.BrokerSimulationMain

Terminal 5: Painel RMI

java -cp "$JAVA_CORE_CP" br.senai.corretora.MonitoringPanelMain

6. O que deve aparecer

- No gateway: mensagens dizendo que as corretoras conectaram e ordens foram encaminhadas
- No core: mensagens de ordens recebidas e trades
- Na custodia: broker inscrito no streaming
- No painel: resumo do mercado com valores negociados


7. Observacao

Se fechar o terminal, o processo daquele modulo para.
No modo `DemoMain`, ele executa a demonstracao e encerra sozinho.
